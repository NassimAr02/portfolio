Veille Technologique : Electron.js

Qu'est-ce qu'Electron.js ?

Electron.js est un framework open-source qui permet de développer des applications desktop multiplateformes en utilisant des technologies web (HTML, CSS, JavaScript). Il est basé sur Node.js et Chromium, ce qui permet aux développeurs de créer des applications riches en fonctionnalités pour Windows, macOS et Linux.

Architecture d'Electron.js

Pourquoi Electron.js ?

Electron.js est largement utilisé pour des applications populaires comme Visual Studio Code, Slack, et Discord. Il permet de réutiliser des compétences en développement web pour créer des applications desktop, ce qui en fait un outil puissant et polyvalent.

Comparatif Electron.js vs .NET C#

Bien que ces deux technologies permettent de créer des applications desktop, leurs approches diffèrent considérablement. Voici une analyse comparative basée sur différents aspects de développement :

Création et gestion de base de données SQLite

En C#, la création de base de données SQLite nécessite l'importation d'un package NuGet, suivie de l'utilisation de méthodes directes pour la création et l'interaction avec la base de données. Les requêtes sont exécutées dans le programme principal en utilisant des méthodes de classe.

Electron.js, quant à lui, utilise des fonctions asynchrones pour interagir avec la base de données en dehors du programme principal. Cette architecture à plusieurs couches diffère de l'approche plus directe de C#.

Génération d'exécutables

Avec .NET C#, la configuration pour générer un exécutable se fait principalement via une interface graphique. Il suffit de passer en mode Release pour générer la solution, ce qui crée automatiquement l'exécutable dans le dossier \bin\Release\.

Pour Electron.js, le processus repose sur des scripts et des configurations dans le fichier package.json. Récemment, certaines fonctionnalités de personnalisation (comme l'ajout d'icônes via console PowerShell) ont été modifiées, nécessitant des approches alternatives.

Configuration d'application

Les deux frameworks utilisent des fichiers de configuration différents mais comparables : le fichier .csproj pour .NET C# et le package.json pour Electron.js. Ces fichiers définissent les informations de l'application et les modules à importer.

Cas d'utilisation appropriés

Electron.js est préférable quand :

  • L'équipe possède déjà des compétences en développement web
  • Le déploiement multiplateforme est une priorité
  • Le développement rapide d'un MVP est nécessaire
  • L'application nécessite une intégration facile avec des services web

.NET C# est préférable quand :

  • La performance est critique
  • L'application nécessite une intégration étroite avec Windows
  • L'équipe a déjà de l'expérience avec l'écosystème .NET
  • L'application doit gérer des ressources système de manière efficace
